BaseNode/TextNode Icon System

Solid circle icons with ring effects for nodes that have children.

Icon Container Classes

  • node-icon: Use for text nodes and other general node types (14x14px circle)
  • task-icon: Use specifically for task nodes with state indicators
  • code-icon: Use specifically for code snippet nodes with syntax highlighting
  • ai-icon: Use specifically for AI chat nodes with square design

Important: Text nodes should use node-icon + text-circle, not task-icon wrapper classes.

Technical Implementation Details

Punch-hole Effect

CSS Mask Implementation

  • 16x16px filled square with primary color (hsl(200 40% 45% / 200 40% 55%))
  • 0.5px border-radius for subtle rounded corners
  • 14x14px "AI" text with punch-hole effect (1px spacing on each side)
  • Background color shows through AI letters (like task checkmarks)
  • Cross-browser compatibility with -webkit-mask fallback
Element Size Purpose
AI Icon Container 20x20px Standard icon container size
AI Square (regular) 16x16px + 0.5px radius Filled square with punch-hole AI text
AI Square (with children) 16x16px inner + 20x20px ring Filled square with 50% opacity ring
Ring Border 2px + 1px radius Semi-transparent square ring effect
Design Rationale
  • Square Shape: Distinguishes AI nodes from circular text/task nodes
  • Punch-hole Text: "AI" letters cut out to show background, similar to task checkmarks
  • Bold 'A' vs Thin 'I': Original icon design maintains visual hierarchy
  • Consistent Sizing: 16x16 active area within 20x20 container matches other icons
  • Theme Adaptive: Works in both light and dark modes automatically

Technical Implementation Details

CSS-Based Rendering (Matches CodeNode Pattern)

Quote Icon Specifications

  • 16x16px filled circle (identical to CodeNode base)
  • 2px offset positioning in 20x20px container
  • Background color: hsl(var(--node-text)) - blue-gray from design system (200 40% 45% / 200 40% 55%)

Design Rationale
  • Identical to CodeNode: Uses exact same base pattern (16x16 circle, 2px offset)

  • Ring Effect: Indicates nested quotes (parent nodes with children)
  • Background Color Contrast: Light/dark theme-specific quotation mark colors
  • Consistent Sizing: Same 20x20 container, 16x16 circle as CodeNode
  • Theme Adaptive: Separate SVG data URLs for light and dark modes

Code Block Display

  return fibonacci(n - 1) + fibonacci(n - 2);

Technical Implementation Details

CSS-Based Rendering (Matches QuoteNode Pattern)

Ordered List Icon Specifications

  • 16x16px filled circle (identical to QuoteNode and CodeNode base)
  • 2px offset positioning in 20x20px container
  • Background color: hsl(var(--node-text)) - blue-gray from design system (200 40% 45% / 200 40% 55%)
  • Numbered list icon: Material Design format_list_numbered icon
  • Icon color: Background color (#fcfcfc light / #252523 dark)
  • No ring effect: canHaveChildren: false (leaf node only)
Element Size Purpose
Ordered List Icon Container 20x20px Standard icon container size
Ordered List Circle 16x16px Solid filled circle, 2px offset in container
Icon Glyph 16x16px SVG Material Design numbered list icon via SVG data URL
Background Color hsl(var(--node-text)) Blue-gray matching design system (200 40% 45% / 200 40% 55%)
Design Rationale
  • Identical to QuoteNode: Uses exact same base pattern (16x16 circle, 2px offset)
  • Material Design Icon: format_list_numbered provides universal recognition for ordered lists
  • CSS Pseudo-Element: SVG data URL in ::after ensures consistent rendering
  • No Ring Effect: canHaveChildren: false means no parent state needed
  • Background Color Contrast: Light/dark theme-specific icon colors
  • Consistent Sizing: Same 20x20 container, 16x16 circle as QuoteNode and CodeNode
  • Theme Adaptive: Separate SVG data URLs for light and dark modes
  • CSS Counter Integration: Icon pairs with CSS counter-based auto-numbering (1, 2, 3...)

Auto-Numbering Display

CSS Counter System

Numbering Specifications

  • Content stores "1. " prefix for all items (markdown convention)
  • View mode strips prefix and displays CSS counter numbers
  • Sequential numbering: 1, 2, 3, 4... (automatic)
  • Counter resets when sequence broken by non-list nodes
  • Number style: font-weight 600, muted-foreground color
  • Number positioning: absolute left, min-width 1.5rem, right-aligned

Cross-Browser Rendering Behavior

SVG Circle Rendering Notes

Universal Rendering Solution

  • Single implementation works consistently across all browsers
  • Hardware acceleration prevents Safari's aggressive anti-aliasing
  • All circles render at exactly 16x16 pixels
  • Parent rings fit perfectly within 20x20 containers without clipping
Browser Rendering Approach CSS Fix Applied
Chrome/Edge Larger radius (r=8) to achieve same 16x16 visual Dual-circle with browser detection
Firefox Crisp edges without positioning offset shape-rendering: crispEdges
Safari Hardware acceleration optimization will-change: transform

Technical Solution: Universal dimensions with hardware acceleration (transform: translateZ(0)) prevent Safari's aggressive anti-aliasing. All circles use r="8" for 16x16 rendering, parent rings use r="9" with stroke-width="2" for perfect 20x20 boundaries. Integer positioning (cx="10" cy="10") eliminates sub-pixel rendering inconsistencies.
