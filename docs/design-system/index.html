Design Principles

🎯 Purpose-Driven

Every design decision serves the user's knowledge management goals. Clean interfaces that get out of the way of thinking and learning.

🤝 AI-Human Collaboration

Seamless integration between human input and AI assistance. Visual patterns that distinguish but don't separate AI-generated and human-created content.

♿ Accessible by Default

WCAG 2.1 AA compliance built into every component. High contrast, keyboard navigation, and screen reader support are non-negotiable.

🔧 Developer Experience

TypeScript-first design tokens, comprehensive documentation, and clear component APIs that make building consistent interfaces effortless.

Design System Architecture

NodeSpace uses a hybrid design system combining shadcn-svelte professional UI components with custom NodeSpace-specific components for AI-native knowledge management.

🏗️ Tier 1: shadcn-svelte Foundation

Professional, accessible UI components (Button, Input, Card, Dialog, etc.)

  • Industry-standard design patterns
  • Copy-paste approach for full control
  • Battle-tested accessibility
  • Unified color system

🧠 Tier 2: NodeSpace Domain Components

Custom components for knowledge management (BaseNode, TextNode, etc.)

  • AI-specific interaction patterns
  • Hierarchical node displays
  • Built on shadcn-svelte foundation
  • Domain-specific functionality

Usage Guidelines

Getting Started with shadcn-svelte

NodeSpace uses professional shadcn-svelte components as the foundation. Import and use these components directly for consistent, accessible UI.

// Import shadcn-svelte components
<script lang="ts">
  import { Button } from "$lib/components/ui/button";
  import { Input } from "$lib/components/ui/input";
  import { Card, CardContent, CardHeader, CardTitle } from "$lib/components/ui/card";
  import { cn } from "$lib/utils.js";
</script>

// Professional UI with shadcn-svelte styling
<Card>
  <CardHeader>
    <CardTitle>Settings</CardTitle>
  </CardHeader>
  <CardContent class="space-y-4">
    <Input placeholder="Enter text..." />
    <Button variant="default">Save Changes</Button>
  </CardContent>
</Card>

Unified Color System

Use shadcn-svelte color variables for consistency. Your NodeSpace colors are automatically mapped:

/* Use shadcn-svelte color variables */
.my-component {
  background-color: hsl(var(--card));
  color: hsl(var(--card-foreground));
  border: 1px solid hsl(var(--border));
  border-radius: var(--radius);
}

/* NodeSpace color system (mapped to shadcn-svelte) */
--primary: 210 100% 40%;           /* Professional blue #007acc */
--background: 0 0% 100%;           /* Clean white backgrounds */
--foreground: 220 13% 10%;         /* Dark text #1a1a1a */
--muted: 210 40% 98%;              /* Light gray #f8f9fa */
--border: 214 32% 91%;             /* Light borders #e1e5e9 */

Building Custom Components

For NodeSpace-specific features, build custom components using the shadcn-svelte foundation:

// Custom component using shadcn-svelte foundation
<script lang="ts">
  import { Card, CardContent } from "$lib/components/ui/card";
  import { Button } from "$lib/components/ui/button";
  import { cn } from "$lib/utils.js";
  
  export let nodeId: string;
  export let hasChildren: boolean = false;
</script>

<Card class={cn("text-node", hasChildren && "has-children")}>
  <CardContent>
    <!-- Custom NodeSpace logic with professional foundation -->
    <Button variant="ghost">🤖 AI Assist</Button>
  </CardContent>
</Card>

Accessibility

NodeSpace design system prioritizes accessibility with WCAG 2.1 AA compliance built into every component.

🎨 Color Contrast

All color combinations meet minimum 4.5:1 contrast ratios for normal text and 3:1 for large text.

⌨️ Keyboard Navigation

Full keyboard navigation support with visible focus indicators and logical tab order.

📖 Screen Reader Support

Proper ARIA labels, roles, and semantic HTML structure for assistive technologies.

🎯 Focus Management

Clear focus indicators and proper focus management for modal dialogs and dynamic content.
