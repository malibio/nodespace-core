Multi-Pane System

Up to 3 content panes for side-by-side viewing. Each pane supports multiple tabs with browser-like behavior. Active pane indicated by subtle warm gray accent.

Browser-Like Tabs

Tab management per pane. Single tab shows no tab bar. Multiple tabs appear with truncated titles (~25 chars). Cmd+click opens in new tab.

Contextual Sidebar

280px fixed sidebar with organized navigation: Daily Journal, Dashboard, Search, Favorites, Recent, Library structure, and Team collaboration.

AI-Native Node System

NodeSpace supports 5 core node types, each with distinct visual indicators and behaviors for comprehensive knowledge management.

Text Nodes

Foundation content blocks. Markdown support, hierarchical structure, collapsible sections. Perfect alignment with warm gray circles.

Task Nodes

Actionable items. Three states: â˜ pending, â—’ in-progress, âœ“ completed. Due dates, assignees, priority ordering.

AI Chat Nodes

Conversational AI interface. Context-aware responses, intent classification, smart linking. Consistent warm gray design with AI sparkle indicator.

Entity Nodes

Structured data records. Custom entity types (Customer, Invoice, Project). Field validation, calculated fields, cross-entity relationships.

Query Nodes

Live data views. Real-time queries with auto-refresh. Filter and sort controls, complex cross-node analysis and reporting.

@Mention System

Universal references. Link to any node or create new ones on-demand. Smart autocomplete with fuzzy search across all content.

Design Principles

Clean Warm Gray Aesthetic

Professional warm gray (hsl(25 25% 55%)) with consistent application across interface elements. Clean, modern design that reduces cognitive load during knowledge work.

AI-Human Collaboration

Seamless integration between human input and AI assistance. Visual patterns that distinguish but don't separate AI-generated and human-created content.

Mathematical Precision

Perfect circle alignment, precise indentation, consistent spacing. Every visual element calculated for optimal readability and hierarchical clarity.

Accessible by Default

WCAG 2.1 AA compliance built into every component. High contrast, keyboard navigation, and screen reader support are non-negotiable.

Usage Guidelines

Getting Started with shadcn-svelte

NodeSpace uses professional shadcn-svelte components as the foundation. Import and use these components directly for consistent, accessible UI.

// Import shadcn-svelte components
<script lang="ts">
  import { Button } from "$lib/components/ui/button";
  import { Input } from "$lib/components/ui/input";
  import { Card, CardContent, CardHeader, CardTitle } from "$lib/components/ui/card";
  import { cn } from "$lib/utils.js";
</script>

// Professional UI with shadcn-svelte styling
<Card>
  <CardHeader>
    <CardTitle>Settings</CardTitle>
  </CardHeader>
  <CardContent class="space-y-4">
    <Input placeholder="Enter text..." />
    <Button variant="default">Save Changes</Button>
  </CardContent>
</Card>

Unified Color System

Use shadcn-svelte color variables for consistency. Your NodeSpace colors are automatically mapped:

/* Use shadcn-svelte color variables */
.my-component {
  background-color: hsl(var(--card));
  color: hsl(var(--card-foreground));
  border: 1px solid hsl(var(--border));
  border-radius: var(--radius);
}

/* NodeSpace color system (mapped to shadcn-svelte) */
--primary: 25 25% 55%;             /* Professional warm gray */
--background: 0 0% 100%;           /* Clean white backgrounds */
--foreground: 220 13% 10%;         /* Dark text #1a1a1a */
--muted: 210 40% 98%;              /* Light gray #f8f9fa */
--border: 214 32% 91%;             /* Light borders #e1e5e9 */
