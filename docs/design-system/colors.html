Usage Guidelines

Do's and Don'ts

‚úÖ Do

  • Use semantic color names (primary, secondary, surface)
  • Test color combinations for accessibility
  • Use node type colors consistently
  • Rely on CSS custom properties for theming
  • Consider both light and dark themes

‚ùå Don't

  • Hard-code hex values in components
  • Use colors below minimum contrast ratios
  • Mix node type colors arbitrarily
  • Override system theme preferences
  • Rely solely on color to convey information

Code Examples

/* Preferred: shadcn-svelte semantic color usage */
.notification {
  background-color: hsl(var(--card));
  color: hsl(var(--card-foreground));
  border: 1px solid hsl(var(--border));
  border-left: 4px solid hsl(var(--primary));
  border-radius: var(--radius);
}

/* NodeSpace extension colors */
.text-node {
  background-color: hsl(var(--card));
  border-left: 4px solid hsl(var(--node-text));
}

/* Avoid: Hard-coded values that won't theme */
.notification {
  background-color: #ffffff;  /* ‚ùå Won't adapt to dark theme */
  border-left: 4px solid #007acc;  /* ‚ùå Hard-coded */
  color: #1a1a1a;  /* ‚ùå No theme support */
}
